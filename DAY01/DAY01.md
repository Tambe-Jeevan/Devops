## Day 1: The Absolute Basics of Servers and the Command Line


[Image of client server architecture]


### Hour 1: Understanding Servers (Theory - 60 Minutes)
* **What is a Server:** A server is just a powerful, remote computer that runs 24/7. It usually does not have a monitor or a mouse attached. It simply waits for requests over the internet and sends data back.
* **The Client-Server Model:** When you open a web app on your phone (the Client), it sends a request over the internet to a Server. The Server then sends back the HTML, CSS, and JavaScript files so your screen can load the app.
* **IP Addresses:** Every server has a unique public IP address (like a phone number) so your computer knows exactly who to call over the vast internet.
* **Your Tech Support Connection:** When a user in your office says a website is not loading, it usually means their client computer cannot successfully route its request to the destination server.

### Hour 2: Getting into the Terminal (Practical - 60 Minutes)
* **Why Command Line?** 90% of Cloud servers (on AWS or Azure) run on the Linux operating system, not Windows. In DevOps, you rarely click icons with a mouse. You type commands to control servers. 
* **The Terminal:** This is the black text window where you type commands to tell the computer hardware what to do.
* **Navigation Command:** `pwd` stands for Print Working Directory. It tells you the exact folder path you are currently sitting inside.
* **Viewing Command:** `dir` (for Windows) or `ls` (for Linux) lists all the files and folders in your current location.
* **Creation Command:** `mkdir` stands for Make Directory. It creates a brand new folder instantly.

## Today's Real-World Practice Project

To be a DevOps engineer, you must get comfortable controlling a machine using only your keyboard. Today, we will simulate this on your local machine.

* Open your Windows computer.
* Click the Start Menu, type **cmd**, and open the Command Prompt application.
* Type `mkdir MyFirstDevOpsProject` and press Enter to create a new folder.
* Type `cd MyFirstDevOpsProject` and press Enter to move inside that new folder.
* Type `echo "Day 1 Complete" > status.txt` and press Enter. This creates a text file and writes words inside it simultaneously.
* Type `dir` and press Enter to verify your new file exists.

You just gave direct, automated commands to your operating system without using a mouse. This is the foundational mindset needed for managing thousands of cloud servers!
