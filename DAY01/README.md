## 🟢 Day 1: Introduction to Linux (DevOps Foundation)

### 1️⃣ Concept: Why Linux for DevOps?
Most DevOps infrastructure runs on Linux distributions rather than Windows.
* **Popular Distros:** Ubuntu, CentOS, RedHat, Debian, Amazon Linux.
* **Why Linux?** Open source, lightweight, secure, powerful CLI, and built for automation.

**The DevOps Workflow Context:**
`Developer` → `Git` → `CI/CD` → `Docker` → `Kubernetes` → `Cloud`
*Linux is the engine that manages servers throughout this entire pipeline.*

### 2️⃣ Environment Setup
To practice, ensure you have one of the following installed:
* **Option 1 (Recommended):** Ubuntu via VirtualBox.
* **Option 2:** Windows Subsystem for Linux (WSL) with Ubuntu 22.04.

### 3️⃣ Basic Linux Commands (Practice)
Open your terminal and master these essential commands:

| Command | Action |
| :--- | :--- |
| `pwd` | Print Working Directory (Where am I?) |
| `ls` | List files in current folder |
| `ls -a` | Show all files (including hidden ones) |
| `cd <folder>` | Change directory |
| `cd ..` | Go back one level |
| `mkdir <name>` | Create a new folder |
| `touch <name>` | Create a new empty file |
| `cat <file>` | Display file content |
| `clear` | Clear the terminal screen |

### 4️⃣ Practice Task: File System Navigation
Perform these steps in order:
1. Create a folder: `mkdir devops_practice`
2. Enter the folder: `cd devops_practice`
3. Create three files: `touch file1.txt file2.txt file3.txt`
4. Verify they exist: `ls`
5. Confirm your path: `pwd`
