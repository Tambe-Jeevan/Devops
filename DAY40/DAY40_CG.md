# 1️⃣ What is Monitoring?

Monitoring means continuously checking the health of systems.

Example:

```text
Server
  ↓
CPU Usage
Memory Usage
Disk Usage
Network Usage
```

You collect and monitor these values.

---

# Real Example

Suppose:

```text
CPU Usage = 95%
```

Without monitoring:

❌ Users complain first

With monitoring:

✅ Alert generated automatically

---

# 2️⃣ What is Observability?

Monitoring tells you:

```text
Something is wrong
```

Observability helps determine:

```text
Why it is wrong
```

---

Three pillars of observability:

```text
Metrics
Logs
Traces
```

---

# 3️⃣ Metrics

Metrics are numerical values.

Examples:

```text
CPU Usage
Memory Usage
Disk Usage
Network Traffic
```

---

Example:

```text
CPU = 25%
Memory = 60%
Disk = 40%
```

These values are monitored continuously.

---

# 4️⃣ Logs

Logs are event records.

Example:

```text
Application Started
Database Connected
User Logged In
Error Occurred
```

---

Linux Log Examples

View system logs:

```bash
sudo journalctl
```

---

View authentication logs:

```bash
sudo tail -f /var/log/auth.log
```

---

View system messages:

```bash
sudo tail -f /var/log/syslog
```

---

# 5️⃣ AWS CloudWatch

One of the most important AWS monitoring services:

Amazon Web Services provides **CloudWatch** for monitoring AWS resources.

CloudWatch monitors:

* EC2
* RDS
* Load Balancers
* Lambda
* Auto Scaling

# 7️⃣ CloudWatch Alarms (🔥 Important)

Create automatic alerts.

Example:

```text
If CPU > 80%
```

Action:

```text
Send Notification
```

---

Production Example:

```text
CPU > 80%
     ↓
CloudWatch Alarm
     ↓
Email Alert
```

---

# 8️⃣ Create a CloudWatch Alarm

Go to:

```text
CloudWatch
 → Alarms
 → Create Alarm
```

---

Choose Metric:

```text
EC2 CPU Utilization
```

---

Condition:

```text
Greater than 80%
```

---

Action:

```text
Send Notification
```

(using SNS)

# 1️⃣2️⃣ Interview Questions

### What is Monitoring?

The process of continuously observing the health and performance of systems.

---

### What is Observability?

The ability to understand the internal state of a system using metrics, logs, and traces.

---

### What is CloudWatch?

AWS monitoring and observability service used to collect metrics, logs, and alarms.

---

### What are Metrics?

Numerical measurements such as CPU usage, memory usage, and network traffic.

---

### What are Logs?

Records of events generated by applications and operating systems.
