## 📦 Day 5: Installing Software & Managing Services

### 📖 Concept
DevOps engineers don't use "Add/Remove Programs." We use **Package Managers (apt)** to install software and **Service Managers (systemctl)** to keep applications running in the background. Background programs are often called "Services" or "Daemons."

### 🛠️ Hands-on Practice

**1. Setup:**
```bash
# Update the local software catalog first
sudo apt update
# Install Nginx (a famous web server)
sudo apt install nginx -y
```

**2. Manipulate:**
```bash
# Control the background service
sudo systemctl status nginx    # Check if it's running (Press 'q' to exit)
sudo systemctl stop nginx      # Shut it down
sudo systemctl start nginx     # Fire it up again
sudo systemctl restart nginx   # Restart (Common after config changes)
sudo systemctl enable nginx    # Auto-start when server reboots
```

**3. Writing & Reading:**
```bash
# Verify the web server is working locally
curl localhost
# Result: If you see HTML code ("Welcome to nginx"), it's working!

# Verify it fails when stopped
sudo systemctl stop nginx
curl localhost
# Result: Connection refused (As expected)
```
